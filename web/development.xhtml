Development

Wikt is written in the C++ language with the use of Qt4.

Source Code

The source code is maintained in a Git repository on Gitorious. You can clone the repository with:

git clone git://gitorious.org/wikt/wikt.git

The code statistics, such as line count, are generated by Ohloh.

Notebook

  • 2009-07-29: Wikt 0.19.0 release notes
    • Code heavily refactored and simplified. Some parts of the Wikt were moved to separate programs. Created a shared library libwikt.
    New Features
    • The lookup panel receives focus when an appropriate key is pressed in the application.
    • Whole application is localizable.
    • Application can be installed in any subdirectory (/opt, /usr, ~, ...)
  • 2009-07-29: convdict23 is incredibly slow on the 20090711 dictionary, because of some category templates (evaluation log for one entry more than 1 GB of text). It seems it is not an error, but the templates do a lot of work. Maybe we should cache the evaluation results. The question is how to elegantly determine what should be cached, because templates containing PAGENAME and similar templates certainly cannot be cached. Or we could try to cache only the evaluation of a single entry.
  • 2009-07-28: Code has been separated into a library, several tools and main application. Some work remains to be done: (1) make tests runnable, (2) better path to library headers, (3) tools should properly parse the command line parameters
